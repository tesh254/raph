Usage docs

Local-first graph memory for coding agents.

raph indexes code, docs, and scoped memory into a local graph, then serves that graph over MCP and Studio so agents can search, traverse, and keep context close to the repo.

Start here MCP setup Open Studio
local first

Graph lives under ~/.raph/

Single binary. Embedded database. No remote graph service needed to get started.

agent ready

MCP tools expose memory + graph traversal

Agents can store scoped memory, search semantically, crawl docs, index the current codebase, and walk node neighborhoods.

studio

Inspect graph with live UI

Explore nodes, edges, metadata, search, SQLite tables, and maintenance actions in Studio at localhost:4545.

Start

One-time config, then index your repo, start MCP, and open Studio.

Install
curl -fsSL https://raw.githubusercontent.com/tesh254/raph/main/install.sh | sh
Bootstrap
raph config init
export OPENROUTER_API_KEY="your-key"
raph init --path .
Optional installs
brew install --cask tesh254/raph/raph
irm https://raw.githubusercontent.com/tesh254/raph/main/install.ps1 | iex
Tip: disable auto-update checks with RAPH_NO_AUTO_UPDATE=1 if you want fully manual release control.

MCP tools

Coding agents connected over stdio can use these tools against local storage.

Memory and search

memory_store records scoped facts, preferences, procedures, and project knowledge with lifecycle metadata, tags, revision, source, and writer details. memory_delete retires active memories by scope and key.

hybrid_semantic_search, multi_query_search, and best_vector_match search memory, code, and documentation with keyword fallback when embeddings are unavailable.

Code and docs

index_codebase defaults to the MCP server working directory, replaces existing nodes for that codebase, and can skip remote embeddings with no_embeddings.

crawl_url fetches one URL. crawl_website returns compact relevant excerpts, with bounded result and excerpt limits. graph_neighbors expands related graph nodes.

MCP setup

New command: raph agents mcp setup --path .. It checks whether each agent is installed, then writes project-scoped MCP config for whichever agents are present. It still stages config for missing ones so setup stays ready.

Command
raph agents mcp setup --path .
raph agents mcp setup --path . --dry-run

Config layout

Each agent wants the same MCP payload, but each stores it in a different file and format.

OpenCode Config under mcp in opencode.json. 
{
  "$schema": "https://opencode.ai/config.json",
  "mcp": {
    "raph": {
      "type": "local",
      "command": ["raph", "start"],
      "enabled": true
    }
  }
}

Studio

Studio is the local graph explorer. It keeps node details live as you click, shows memory and crawl metadata, supports search and neighbor expansion, and includes local-only database maintenance actions.

Run
raph studio --port 4545
Open http://localhost:4545.
Graph behavior

All nodes stay visible, node clicks refresh properties, links stay interconnected, and table inspection is capped to keep the page responsive.

Maintenance actions

The init action clears local graph data, indexes the current workspace, and crawls the configured seed URL. The clear action wipes local graph data. Use Studio only on trusted local machines.

Troubleshooting
raph --verbose studio --port 4545

Troubleshooting

No embeddings

Run raph config init, then set OPENROUTER_API_KEY. Without it, raph falls back to keyword search.

Need a fresh MCP install

Run raph agents mcp setup --path . --dry-run first to preview file targets, then rerun without --dry-run.
